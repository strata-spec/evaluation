### q012 · multi_table · BASELINE PASS / STRATA FAIL

**Question:** How many distinct people have been credited as Director?

**Baseline SQL:**
```sql
SELECT COUNT(DISTINCT person_id)
FROM casts c
JOIN jobs j ON c.job_id = j.id
WHERE j.name = 'Director';
```

**Strata SQL:**
```sql
SELECT COUNT(DISTINCT p.id)
FROM people p
JOIN cast_crew cc ON p.id = cc.person_id
JOIN jobs j ON cc.job_id = j.id
WHERE j.name = 'Director';
```

**Reference SQL:**
```sql
SELECT COUNT(DISTINCT c.person_id) FROM casts c JOIN jobs j ON c.job_id = j.id WHERE j.name = 'Director'
```

**Failure modes:** Strata: schema hallucination

**Notes:** Requires casts JOIN jobs on job_id = id. Agent must infer the join key. 'Director' exists as jobs.name (id=21). Agent might try to filter casts.role instead of joining jobs.


### q014 · multi_table · BASELINE FAIL / STRATA FAIL

**Question:** What are the 10 most commonly assigned keywords for movies?

**Baseline SQL:**
```sql
SELECT c.name, COUNT(*) as frequency
FROM movie_keywords mk
JOIN categories c ON mk.category_id = c.id
GROUP BY c.name
ORDER BY frequency DESC
LIMIT 10;
```

**Strata SQL:**
```sql
SELECT k.name, COUNT(*) as assignment_count
FROM keywords k
JOIN movie_keywords mk ON k.id = mk.keyword_id
GROUP BY k.id, k.name
ORDER BY assignment_count DESC
LIMIT 10;
```

**Reference SQL:**
```sql
SELECT c.name, COUNT(*) as cnt FROM movie_keywords mk JOIN categories c ON mk.category_id = c.id GROUP BY c.name ORDER BY cnt DESC LIMIT 10
```

**Failure modes:** Baseline: wrong table or column, Strata: schema hallucination

**Notes:** Requires movie_keywords JOIN categories on category_id = id. Keywords are stored in the categories tree under root_id=8 (Keyword). Agent might confuse movie_keywords with movie_categories (genres) since both reference the categories table.


### q015 · multi_table · BASELINE PASS / STRATA FAIL

**Question:** How many distinct people have cast credits in movies released after 2010?

**Baseline SQL:**
```sql
SELECT COUNT(DISTINCT c.person_id)
FROM casts c
JOIN movies m ON c.movie_id = m.id
WHERE EXTRACT(YEAR FROM m.date) > 2010;
```

**Strata SQL:**
```sql
SELECT COUNT(DISTINCT p.id)
FROM people p
JOIN cast_credits cc ON p.id = cc.person_id
JOIN movies m ON cc.movie_id = m.id
WHERE EXTRACT(YEAR FROM m.date) > 2010;
```

**Reference SQL:**
```sql
SELECT COUNT(DISTINCT c.person_id) FROM casts c JOIN movies m ON c.movie_id = m.id WHERE m.date >= '2011-01-01'
```

**Failure modes:** Strata: schema hallucination

**Notes:** Requires casts JOIN movies on movie_id = id. Date filter forces the join — casts has no date column. Tests the most common FK relationship in the schema.
